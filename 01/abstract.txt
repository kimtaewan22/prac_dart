1. 변수를 만드는 방법 2가지
타입을 적고 변수명 쓰기 or var를 쓰고 변수명 쓰기
dart에서는 최대한 var키워드를 사용하는 것을 권장함
-> 타입을 사용하는 방식은 클래스의 property를 사용하는 경우 권장
메서드 내에서 선언하는 변수의 경우 컴파일러가 변수 타입을 추론하므로 var를 사용

2. 변수에 값을 단 한번만 할당하고 싶은 경우 var대신 final키워드 사용

3. dynamic은 어떤 데이터가 들어올지 모른다고 말해주는 키워드

4. const 우리가 컴파일 할 때 값을 알고 있는 변수를 만드는 것. 핵심은 컴파일 전에 값을 알고있어야 한다는 점이며, 당연히 수정 불가능

5. final은 런타임 중에 만들어 질 수 있다는 점이 const와의 차이점

6. null safety
-> 기본적으로 dart의 모든 변수는 non-nullable타입
-> nullable타입을 만드려면 자료형 뒤 ?을 붙여주면 됨
-> 어떤 변수가 null일 수도 있다는 것을 알려주는 것이다.

7. late는 final, var 등에 붙여줄 수 있는 키워드
-> 아직은 어떤 데이터가 들어갈 지 모른다고 말해주는 키워드
-> 즉, 나중에 정의하기로 했다는 것을 알려주며, 보통 API를 통해 값을 가져올 때 많이 사용함, 
-> 아직 정의되지 않은 late에는 접근할 수 없음
